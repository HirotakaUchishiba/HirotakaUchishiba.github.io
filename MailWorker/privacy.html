Privacy Policy

This policy explains how MailWorker, developed by Hirotaka Uchishiba, accesses, uses, stores, shares, and protects Google user data.

1. Data MailWorker accesses

After the user grants Google OAuth permission, MailWorker may access the following data for the features the user chooses to run:

  • Gmail message and thread identifiers, sender, subject, received time, selected headers, labels, attachment filenames, and plain-text message content for messages matching configured searches and cleanup rules.
  • Gmail message and filter controls needed to label a message, move it to Trash, permanently delete it after the retention rule, send a standard email unsubscribe request, or create a sender filter.
  • Google Calendar event identifiers, titles, dates and times, locations, and descriptions in the calendar and date range selected by the user.

MailWorker does not access Google Contacts.

2. How Google user data is used

Google user data is used only to provide the user-facing functions shown in MailWorker: previewing and applying mailbox organization rules, identifying unwanted mail, preparing calendar event proposals from email, creating approved events, finding possible duplicate events, and applying approved event updates or deletions.

Messages matching a configured high-confidence unwanted-mail decision or deterministic cleanup rule may be moved to Trash automatically within per-run and daily limits. Standard List-Unsubscribe mail requests and sender-filter creation also run automatically when the configured classification and confidence conditions are met; MailWorker does not open arbitrary unsubscribe links.

MailWorker records when it moved a message to Trash or first observed another message in Trash. If the message remains in Trash for at least 24 hours, MailWorker permanently deletes it during the next manual or scheduled cleanup. Permanent deletion is irreversible. Duplicate-event changes require explicit approval. Calendar creation also requires approval unless the user has enabled the optional trusted-sender automation and its confidence and daily-limit conditions are met.

3. Local storage and retention

The Google OAuth Desktop client secret, access token, and refresh token are stored in the user's macOS Keychain. The client secret is provisioned locally and is not included in the app bundle. Claude subscription authentication remains under Claude Code's control and is not copied into MailWorker. Preferences, Trash observation times, calendar approval queues, message-to-event mappings, and audit records are stored only in MailWorker's sandboxed Application Support directory on the user's Mac.

Local app data remains until the user removes it or removes the app's data. OAuth tokens remain until the user disconnects Google in MailWorker, revokes access in the Google Account settings, or removes the related Keychain items. MailWorker does not operate a developer server that stores Google user data.

4. Data sharing and transfer

MailWorker communicates directly from the user's Mac with Google OAuth, Gmail API, and Google Calendar API. It does not sell Google user data, use it for advertising, or disclose it to data brokers.

If the user runs an LLM-assisted feature, MailWorker places the message sender, subject, received time, up to 12,000 characters of plain-text content, and whether a List-Unsubscribe header exists in a private temporary queue on the user's Mac. The bundled local bridge passes those fields to the official Claude Code CLI using the user's Claude subscription. It does not send the Gmail message identifier. Tools and session persistence are disabled, and queue files are removed after processing. Anthropic processes the submitted data under the user's agreement with Anthropic and its applicable privacy and retention terms.

MailWorker does not use Google user data to train generalized or personalized AI models.

5. Security

MailWorker uses OAuth authorization code flow with PKCE, a random loopback redirect port, state validation, HTTPS API connections, macOS Keychain, App Sandbox, request limits, calendar approval boundaries, Trash retention tracking, and local audit records. No method of storage or transmission is guaranteed to be completely secure.

6. User choices and deletion

Users can disconnect Google from MailWorker settings and can revoke MailWorker from their Google Account permissions. Users can remove local state by deleting MailWorker's Application Support data and remove credentials through macOS Keychain Access. A message may be restored from Gmail Trash before MailWorker's 24-hour retention period expires; after permanent deletion it cannot be restored. Gmail filters can be removed in Gmail, and Calendar changes can be reviewed in Google Calendar.

7. Google API Services User Data Policy

MailWorker's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

8. Changes and contact

This policy may be updated when MailWorker's data practices change. The effective date above will be updated when changes are published. Questions or deletion requests can be sent to hirotaka1999821@gmail.com.
