Privacy Policy

This policy explains how MailWorker, developed by Hirotaka Uchishiba, accesses, uses, stores, shares, and protects Google user data.

1. Data MailWorker accesses

After the user grants Google OAuth permission, MailWorker may access the following data for the features the user chooses to run:

  • Gmail message identifiers, sender, subject, received time, selected headers, labels, and plain-text message content for messages matching user-defined searches.
  • Gmail message and filter controls needed to archive a message, move it to Trash, send an approved standard email unsubscribe request, or create an approved sender filter.
  • Google Calendar event identifiers, titles, dates and times, locations, and descriptions in the calendar and date range selected by the user.

MailWorker does not access Google Contacts or permanently delete Gmail messages.

2. How Google user data is used

Google user data is used only to provide the user-facing functions shown in MailWorker: previewing and applying mailbox organization rules, identifying unwanted mail, preparing calendar event proposals from email, creating approved events, finding possible duplicate events, and applying approved event updates or deletions.

Trash, unsubscribe, sender-filter, and duplicate-event changes require explicit approval. Calendar creation also requires approval unless the user has enabled the optional trusted-sender automation and its confidence and daily-limit conditions are met.

3. Local storage and retention

Google OAuth access and refresh tokens and the user's LLM API key are stored in the user's macOS Keychain. Preferences, approval queues, message-to-event mappings, and audit records are stored only in MailWorker's sandboxed Application Support directory on the user's Mac.

Local app data remains until the user removes it or removes the app's data. OAuth tokens remain until the user disconnects Google in MailWorker, revokes access in the Google Account settings, or removes the related Keychain items. MailWorker does not operate a developer server that stores Google user data.

4. Data sharing and transfer

MailWorker communicates directly from the user's Mac with Google OAuth, Gmail API, and Google Calendar API. It does not sell Google user data, use it for advertising, or disclose it to data brokers.

If the user configures and runs an LLM-assisted feature, MailWorker sends the message sender, subject, received time, up to 12,000 characters of plain-text content, and whether a List-Unsubscribe header exists directly to the OpenAI API using the user's own API key. It does not send the Gmail message identifier. Requests set store=false. OpenAI processes that data under the user's agreement with OpenAI and OpenAI's applicable privacy and retention terms.

MailWorker does not use Google user data to train generalized or personalized AI models.

5. Security

MailWorker uses OAuth authorization code flow with PKCE, a random loopback redirect port, state validation, HTTPS API connections, macOS Keychain, App Sandbox, request limits, user approval boundaries, and local audit records. No method of storage or transmission is guaranteed to be completely secure.

6. User choices and deletion

Users can disconnect Google from MailWorker settings and can revoke MailWorker from their Google Account permissions. Users can remove local state by deleting MailWorker's Application Support data and remove credentials through macOS Keychain Access. Removing a Gmail filter or restoring items from Gmail Trash or Archive can be done in Gmail; Calendar changes can be reviewed in Google Calendar.

7. Google API Services User Data Policy

MailWorker's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

8. Changes and contact

This policy may be updated when MailWorker's data practices change. The effective date above will be updated when changes are published. Questions or deletion requests can be sent to hirotaka1999821@gmail.com.
